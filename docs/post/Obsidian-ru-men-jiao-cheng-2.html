  • 笔记越来越多，怎么分类管理？文件夹还是标签？

  • 有没有快捷键可以让我写笔记快一点？
  • 大家都在说的 Obsidian 插件，到底该装哪些？

别急，一个一个来 👇

一、文件夹组织——给知识建个"书架" 📂

大部分人一开始都会用文件夹来管理笔记，比如这样：

我的知识库/
├── 学习笔记/
│   ├── Python.md
│   ├── 数据分析.md
│   └── 机器学习.md
├── 工作相关/
│   ├── 项目A.md
│   └── 周报.md
└── 生活记录/
    └── 读书笔记.md

看起来很清晰对吧？文件夹的优势就是直观、一目了然。

但是问题来了——一篇笔记只能放在一个文件夹里。

假设你写了一篇笔记叫《用Python做数据分析》，它既属于"Python"，也属于"数据分析"。那它该放哪个文件夹？放哪边都不太对。

这时候，标签就派上用场了。

二、标签系统——给笔记"贴标签" 🏷️

Obsidian 的标签很简单，在笔记里写 #标签名 就行：

# 用Python做数据分析

#Python #数据分析 #可视化


今天我们用 Pandas 处理了一份销售数据……

一个笔记可以加多个标签，比如这篇笔记可以同时拥有 #Python、#数据分析、#可视化 三个标签。

然后点击左侧顶部栏的搜索面板（一个类似放大镜的图标），然后选择tag标签🏷，就能看到所有标签和对应的笔记数量：

点击任意标签，就会列出所有带这个标签的笔记。

如果你在设置里面开关闭了这个选项，你就可以直接进行预览文件

这样，以后就可以快速对照来打开需要的文件

标签的好处：

  • ✅ 一篇笔记可以有多个标签，灵活分类
  • ✅ 支持嵌套标签，比如 #编程/Python、#编程/JavaScript
  • ✅ 适合按"主题"或"属性"来归类

三、文件夹 vs 标签，到底怎么用？

这是很多新手都会纠结的问题。我的建议很简单：

文件夹管"项目"，标签管"主题"

在知识库的组织方式上，文件夹与标签各有适用场景：

适合文件夹的情况：

  1. 按项目（如项目A、项目B）
  2. 按用途（如日记、读书笔记）分类时

适合使用标签的情况：

  1. 按主题（如Python、数据分析）
  2. 按状态（如待整理、已完成）
  3. 按优先级（如重要、普通）

我的个人实践：

我的知识库/
├── 00-收件箱/        ← 临时笔记，先扔这里
├── 10-学习笔记/      ← 长期知识积累
├── 20-项目笔记/      ← 按项目划分
├── 30-日记与周报/    ← 时间相关
├── 40-读书笔记/      ← 阅读记录
└── 90-归档/          ← 已完成项目

然后在笔记里打标签：#Python、#数据分析、#待整理、#重要……

  • 文件夹保证了大方向不乱
  • 标签提供了跨文件夹的灵活检索

💡 黄金法则： 文件夹层级不超过 3 层，标签数量不设限但要有规律命名。

四、模板功能——高效偷懒的神器 ⚡

你肯定遇到过这种情况：

  • 每天写日记，但每次都要手动敲日期
  • 每周写周报，格式总是调来调去
  • 做读书笔记，每次都要写书名、作者、日期

这时候模板（Templates） 就能帮你一键搞定！

4.1 开启模板功能

  1. 打开 设置 ⚙️ → 核心插件
  2. 找到 模板，打开开关
  3. 设置模板文件夹路径，比如 模板/

4.2 创建你的第一个模板

在 模板/ 文件夹下新建一个笔记，取名为 日记模板.md：

# {{date}}
## 📌 今日重点
- 
## 📝 工作记录
- 
## 💭 思考与感悟
- 
## 📖 今日阅读/学习
-

其中 {{date}} 是 Obsidian 的模板变量，插入时会自动替换为当前日期。

4.3 使用模板

你也可以设置快捷键或者安装 Templater 插件(安装步骤见下文)，在任意笔记中，按 Ctrl + P 打开命令面板，然后来获得更强大的模板功能。

六、社区插件入门——让Obsidian更强大 🔌

Obsidian 真正厉害的地方在于它的插件生态。目前社区有 2000+ 个插件，几乎能解决你遇到的所有问题。

6.1 如何安装插件

  1. 打开 设置 ⚙️ → 第三方插件 → 社区插件
  2. 点击 浏览，搜索你要的插件
  3. 点击 安装，然后 启用

⚠️ 安全提醒： 尽量安装下载量高、更新频繁的插件。社区插件的代码是开源的，但建议不要安装来源不明的插件。

6.2 插件误区⚠️

网上有许多教程，一上来就告诉你dataview、excalidraw、task等xx插件必装

我告诉你，根本没有必要，有些插件根本没有对应的刚需

Obsidian是个人知识库管理的一个工具，最重要的是找到适合自己、能满足自己实际需求的插件才是所谓的“必装插件”

目前个人做得比较成熟和优秀的插件分为下面几类：

结构化元数据层：Properties、Bases、Dataview、Metadata Menu；
自动化层：Templates、Templater、QuickAdd、Commander；
编辑增强层：Linter、Advanced Tables、Admonition；
可视化层：Canvas、Excalidraw、Advanced Canvas、Mermaid/PlantUML；
研究工作流层：Zotero Integration、Citations、ZotLit、Better BibTeX；
发布导出层：Publish、Pandoc、Webpage HTML Export、Better Export PDF。

不过，正如我所说的，你没必要都安装

下面我给你两个所有Obsidian新手真正需要安装的插件：

6.2.1 必装插件

Custom Attachment Location

  • 功能简介：自定义附件（如图片、文件）的存储位置，支持按笔记所在文件夹或指定路径存放附件，避免默认将所有附件都堆在根目录或 attachment 文件夹下。
  • 安装方式：设置 → 第三方插件 → 社区插件 → 搜索 “Custom Attachment Location” → 安装并启用
  • 主要使用场景：
    1. 希望将笔记中的图片等附件与本笔记放在同一文件夹内，便于整理和迁移。
    2. 为不同项目或主题笔记设置不同的附件路径。
  • 推荐理由：解决附件散乱问题，让知识库更整洁；尤其适合大量插入图片或文件的用户。

Editing Toolbar

  • 功能简介：在笔记编辑区域顶部添加一个可视化的格式工具栏，提供加粗、斜体、标题、列表、引用、代码块等常用 Markdown 格式按钮，无需记忆快捷键或命令。
  • 安装方式：设置 → 第三方插件 → 社区插件 → 搜索 “Editing Toolbar” → 安装并启用
  • 主要使用场景：
    1. 新手不熟悉 Markdown 语法或快捷键，需要直观的点击操作。
    2. 日常快速插入列表、表格、链接等常用格式。
  • 推荐理由：降低入门门槛，提高编辑效率；适合习惯“所见即所得”操作的用户。

没错，就这两个，其他的都是取决于你的需要

6.2.2 画图专用插件

如果你有画图或白板笔记的需求，下面这两个插件也同样值得安装：

Excalidraw

  • 功能简介：在笔记中嵌入手绘风格的白板画板，支持流程图、思维导图、示意图等，可插入图片、添加文字、自由绘图，所有内容都保存在 Markdown 笔记中。
  • 安装方式：设置 → 第三方插件 → 社区插件 → 搜索 “Excalidraw” → 安装并启用
  • 主要使用场景：
    1. 需要快速画流程图、架构图或概念草图。
    2. 在读书笔记或学习笔记中绘制示意图，增强理解。
    3. 团队协作时共享可视化思路。
  • 推荐理由：轻量、直观、完全本地化，绘图内容与笔记深度绑定，无需切换应用。

Better Canvas

  • 功能简介：增强 Obsidian 原生 Canvas（白板）功能，增加更多节点样式、连接线类型、自动布局、导出为图片等，让白板更专业。
  • 安装方式：设置 → 第三方插件 → 社区插件 → 搜索 “Better Canvas” → 安装并启用
  • 主要使用场景：
    1. 使用 Canvas 进行项目规划、知识地图梳理。
    2. 需要更丰富的节点颜色、图标和连线样式。
    3. 希望将白板导出为高清图片分享或打印。
  • 推荐理由：在原生 Canvas 基础上大幅提升功能，满足进阶可视化需求，且与 Obsidian 生态无缝融合。

excalidraw的话，它更适合快速绘制流程图

在这里，可以在画布自由导入PDF、markdown、笔记卡片、latex公式，SVG图像等，适合拿来演讲演示，当然功能远远不止这些

顶部工具栏是图案，按住空格，然后鼠标左键可以进行画布的拖拽和移动，按住ctrl可以选择目标，日常用好这些就够了

其他的功能比如AI接入这些看情况可选

至于绘图的入口也很方便，在文件夹这边空白地方右键即可，弹出对话框就能看到选项，当然左侧工具栏会有显示。

至于canvas（也叫画布、白板），是Obsidian自带的，可以用于自由绘制和连接笔记，但功能相对基础，如果你需要更丰富的绘图能力或手写支持，建议配合专业插件使用。也就是我上面介绍的那个插件，在这里你可以导入excalidraw文件，让图示更方便和更加直接

如果你有同步的需求，我会推荐下面几个同步用的插件，根据情况选择一个适合的即可，目前同步方案主流的是Fast Note Sync、Remotely Save、Nutstore Sync、Git、官方同步，可以从价格、配置难易、同步速度和稳定性根据自身情况进行选择

方案 价格 配置难度 同步速度 稳定性 适用场景
Official Sync (官方同步) 月付/年付（约 $5/月起） 极低（一键开启） 极快（端到端加密） 极高 注重隐私、愿意付费、追求最好体验的用户
Remotely Save 免费 + 自备云存储（如 OneDrive、Dropbox、S3） 中等（需配置云盘 API） 较快（取决于云盘） 较高 希望免费、熟悉云盘配置，能接受偶尔冲突
Fast Note Sync 免费 + 自备 WebDAV 服务 中等（需 WebDAV 地址） 较快 较高 有自建 NAS 或支持 WebDAV 的云盘（如坚果云）
Nutstore Sync 坚果云付费版（约 ¥30/月） + 插件 低（安装插件登录即可） 快 高 国内用户，使用坚果云同步
Git (如 obsidian-git) 免费 + 自备 GitHub/Gitee 仓库 较高（需 Git 基础） 中等（取决于 git push/pull） 高 开发者、习惯版本管理、需要历史记录的用户
OneDrive/Google Drive 直接同步 免费（有容量限制） 低（仅需安装桌面同步客户端） 快（但可能因文件锁定导致冲突） 较低 仅用于备份或单设备临时同步，不推荐多设备同时编辑

我的建议：如果不差钱，直接上官方同步，省心安全；如果想免费且有一点点折腾能力，Windows用户用Remotely Save 搭配 OneDrive最优解；Mac用户使用iCloud；国内用户用坚果云 + Nutstore Sync 插件体验也不错。

Obsidian同步插件使用教程在各自的主页里的配置指南会描述得比较详细，这里我不多赘述，有需要的可以私信问我咨询

6.2.4 Obsidian+AI工作流必装插件

如果你希望在 Obsidian 中直接使用 AI 来辅助写作、整理笔记或与知识库对话，下面两个插件值得一试：

Claudian

  • 功能简介：将 Claude AI 深度集成到 Obsidian，可在笔记内与 AI 对话、提问、总结、翻译、润色、生成内容等，支持引用当前笔记或整个知识库作为对话上下文。
  • 主要使用场景：
    1. 写作时让 AI 帮忙润色、扩写或翻译。
    2. 阅读长文时快速总结要点。
    3. 用自然语言向知识库提问，获取基于笔记内容的回答。
  • 理由：无缝嵌入 Obsidian 工作流，无需切换应用；对话内容可保存为笔记，构建可追溯的 AI 辅助记录。

YOLO

  • 功能简介：新一代 AI 助手插件，提供深度集成 Obsidian 的智能对话体验。支持文件读写、知识库搜索、模板生成、技能扩展等，能直接操作和管理你的 Obsidian 笔记，而不仅仅是对话。
  • 主要使用场景：
    1. 让 AI 帮你整理、归类、重命名笔记。
    2. 批量生成或更新笔记模板。
    3. 在知识库中进行智能搜索和内容分析。
    4. 通过技能系统定制 AI 行为，更贴合个人工作流。
  • 理由：不仅是对话工具，更能直接「操作」你的 Obsidian；强大的技能系统让 AI 成为真正的知识库助手。

由于这个配置比较繁琐，不是开箱即用的插件，对于前期Obsidian基础教学部分，先不做大篇幅展开，后续几期在科研工作流的教程环节会再次提及

📋 本教程核心要点总结表

为了让你一眼回顾整篇教程的精华，我把所有关键概念整理成了下表，方便你对照使用。

模块 总结 适配场景 误区
文件夹 按项目/用途建“书架”，保证大方向不乱 项目笔记、日记、归档 层级过深，导致查找慢
标签 一篇笔记可拥有多个主题标签，灵活跨文件夹检索 按主题、状态、优先级归类 标签命名无规律、数量过多导致冗余
模板 一键插入预设格式，彻底告别重复排版 日记、周报、读书笔记 模板设计过于复杂，反而不灵活
快捷键 记住 Ctrl+N/P/O/E 等，操作速度翻倍 日常编辑、快速切换、搜索 想一次性记住所有快捷键
插件 先装 Custom Attachment Location 和 Editing Toolbar，有刚需再扩展 附件管理、新手编辑辅助 安装过多插件，冲突导致卡顿
同步 根据设备数量、预算、技术能力选择最适合的方案 多设备协同、备份 图便宜用免费云盘直同步，导致笔记损坏或冲突
